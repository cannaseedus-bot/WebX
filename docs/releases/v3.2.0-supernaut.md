## Core Innovations

### Monotonic 7-State Lifecycle

`REGISTERED → VALIDATED → BOUND → ACTIVE → EXECUTED → OBSERVED → ARCHIVED`

`DispatchContext.advance(next_state)` raises `ValueError` if the new state is not strictly after the current state. No rollback, no re-entry. This enforces that every dispatch follows the full pipeline in order.

### DispatchContext

- `token_ids` — input token sequence
- `route` — `ExpertRoute(expert_id, expert_name, addon_shard, strength)`
- `state` — current `LifecycleState`
- `logits` — output from `SCXMoEForward` (populated at EXECUTED state)
- `report` — dict: `forward_rc`, `expert_id`, `expert_name`, `seq_len`, `out_size`

### SkillRunResult

- `level_value` — `min(len(actionsExecuted), 19)` — vigesimal (0-19) score
- `vigesimal_addr` — `round_num - 1` — position in the Mayan fold
- Status: `ok | skipped | error`

### RoundContext

- `vigesimal_addr = round_num - 1` — Mayan base-20 address
- `level_value = min(ok_skills_count, 19)` — round score

### Mayan Vigesimal Fold

```
MayanFold(T) = Σ level_value[n] × 20^n  (n = 0..19)
```

20-round execution accumulates level_values, then computes a single vigesimal fold value. This encodes the quality of all 20 rounds into one number.

### 20-Round Sequential Execution

Stock skills (every round): `as-xcfe-stack-intel`, `codex-agent`, `micronaut`, `micronaut-agent-factory`

Phase-dynamic skills by round range:
- Rounds 1-3: stock only
- Rounds 4-7: +`workflow-orchestrator`, `sql-skill`
- Rounds 8-12: +`matrix-sandbox`, `sql-skill`
- Rounds 13-16: +`agentic-micronaut`, `sql-skill`
- Rounds 17-20: +`bot-builder`, `project-factory`

Each skill completes fully before the next starts. Each round completes fully before the next round begins.

### sxme_host.dll API Surface

| Symbol | Signature | Description |
|--------|-----------|-------------|
| SCXMoEProbeRuntime | `() → int` | Initialize D3D12; 0=OK |
| SCXMoELoadAddonShard | `(path, strength) → int` | Upload adapter tensors to Expert 2 GPU |
| SCXMoEForward | `(tokenIds, seqLen, weightsPath, logits, outSize) → int` | Full MoE forward |
| SCXMoEShutdown | `() → void` | Release GPU resources |

### Sigil Action Parser

`parse_sigil_actions(doc)` walks a sigil document and resolves every `@`-prefixed key to a skill handler. Supports nested dicts with dot-prefixed key names. Control vectors: `@load @route @dispatch @merge @validate @report`. Transitions: `@next @accept @reject`.

## Notes

- The DaemonSession / NumaticManifoldEngine / LLMRouter live in `supernaut_native.exe` (C++), not in the Python orchestrator. The JS port exposes the protocol, not the native runtime.
- The KuhulGeometricState SHM read (`_read_shm`) is Windows-only; JS port consumers should supply telemetry from SyncWorker (Agents.NET port) or mock it.
- `EXPERT_MAP` in lifecycle.js intentionally duplicates the one in expert-registry.js — they're the same 8 experts but lifecycle.js is authoritative for the orchestrator routing layer.

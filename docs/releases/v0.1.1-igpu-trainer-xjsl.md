## What this release added

### XJSL — Cross-platform Shader JSON Language

XJSL treats GPU kernels as first-class JSON nodes. A shader document describes compute
shaders with typed input/output buffers and an embedded WGSL kernel string. The lowering
layer generates both WGSL (for WebGPU) and HLSL (for D3D11/cs_5_0) from a single source
document — deterministically, with no runtime ambiguity.

**Canonical example:** `xjsl/examples/igpu_trainer_kernels.xjsl.json`
- `gpt2_matmul`: workgroup [16,16,1], typed WGSL kernel, f32 inputs + outputs
- `pi_field_propagate`: workgroup [256,1,1], phase-aware field propagation

**Runtime files:**
| File | Purpose |
|------|---------|
| `xjsl/runtime/xjsl-lowering.mjs` | JSON IR → WGSL + HLSL (file-based, Node.js) |
| `xjsl/runtime/xjsl-wgpu-runtime.mjs` | WebGPU dispatch adapter |
| `tools/validate-xjsl.mjs` | Schema enforcement (CLI) |

**Ported to WebX-3D:**
| Source | WebX-3D target |

### Two new HLSL shaders

**`gpt2_lm_head_bwd.hlsl`** — LM head backward pass
- Dispatch(vocab_size, 1, 1), numthreads(768, 1, 1)
- Scatters dL/dwte: `g_wte[v,d] += dlogits[v] * ln_out[last_pos,d]`
- SRV: dlogits[V], ln_out[B*S, E] — UAV: g_wte[V, E]

**`gpt2_adam_wte.hlsl`** — GPU Adam update for wte embedding table
- Dispatch(vocab_size, 1, 1), numthreads(768, 1, 1)
- Full Adam (moment, second moment, bias correction, weight decay)
- Zeros gradient accumulator in-place — no separate clear shader needed

### Hybrid Adam design

CPU Adam handles all parameters except wte. `gpt2_adam_wte.hlsl` runs on GPU
for the embedding table only, since wte accumulates scatter-style gradients
from every token in the batch. The `grad_clip` field was added to `TrainerConfig`
as a scalar global gradient norm clamp applied before both Adam variants.

## XJSL memory model

XJSL adopts the DICT/LANE/TILE model from XCFE:
- **DICT** — JSON objects as symbol tables (shader name → definition)
- **LANE** — JSON arrays as contiguous numeric streams (workgroup sizes, buffer layouts)
- **TILE** — kernel strings as binary payloads (embedded WGSL/HLSL source)

This makes XJSL documents directly memory-addressable at the JSON layer without
a separate compilation pipeline step.

## Autograd rules added

`src/xjsl/autograd.js` implements JS reference implementations of backward
passes for: add, mul, clamp, relu, sigmoid, tanh, gelu, softmax, rmsnorm,
layernorm, reduce_sum, reduce_mean, reduce_max, matmul, dot.

These are used for CPU-side gradient checking (not GPU dispatch). GPU backward
shaders are the authoritative path for training.

## Core Innovations

### sxme_router.hlsl — Top-K Expert Selection

One thread per token. Serial dot-product + top-2 scan (no WaveOps):
1. Compute gate scores: `score[e] = dot(hidden, routerGate[e])`
2. Top-2 selection: serial scan, skip already-selected
3. Softmax over top-2 scores → normalized weights

Thread group: `[numthreads(64, 1, 1)]`. Supports up to 16 experts without dynamic alloc.

### sxme_expert.hlsl — SwiGLU FFN + Expert 2 Adapter

SwiGLU: `gate = silu(Wg·x)`, `up = Wu·x`, `out = Wd·(gate*up)`

**Expert 2 (amplify) adapter injection — AdapterScale():**
1. Project hidden[1024] → 256 via stride-4 mean-pool
2. Cosine similarity against `adapterRouteEmb[adapterN × 256]`
3. `scale = 1 + saturate(bestScore) × adapterRouteBias[bestIdx] × adapterStrength`
4. Apply scale to FFN output

Root signature slots:
- b0: ExpertParams {batchSize, hiddenSize, intermediateSize, topK, numExperts, layerIdx}
- b1: AdapterParams {adapterN, adapterStrength}
- t0: hiddenIn, t1: expertIds, t2: expertWeights
- t3: wGate, t4: wUp, t5: wDown
- t6: adapterRouteEmb [adapterN × 256], t7: adapterRouteBias [adapterN]
- u0: hiddenOut

### Compute Path Priority

1. D3D12 cs_5_0 DXBC (FXC) — HD 4600 compatible
2. WGSL/WGPU fallback
3. NumPy CPU fallback

**Note:** DXC cs_6_0 compiles but `CreateComputePipelineState` fails on Haswell iGPU (E_FAIL). Must use FXC cs_5_0 for HD 4600.

### Weight Format

Per-expert per-layer: `/experts/expert_N/layer{L}_w{gate,up,down}.npy`
Router gate: `/weights/router_gate.npy`

### Training Status (opus46_final.jsonl — 9,633 records)

500-step CPU run: loss 5.28 → 2.34, ~84 min at ~40 tok/s. Embedding + lm_head in SCXMoEForward GPU path are CPU stubs pending GPU wiring.

## WebX-3D Port

**`src/moe/router.js`**
- `routeTopK(hiddenStates, routerGate, opts)` → `{expertIds, expertWeights}`
- `routeToken(hidden, routerGate, numExperts, topK)` → `{experts[], weights[]}`
- Architecture constants: `MOE_NUM_EXPERTS=8`, `MOE_TOP_K=2`, `MOE_HIDDEN_SIZE=1024`, etc.

**`src/moe/expert.js`**
- `expertForward(hidden, wGate, wUp, wDown, hiddenSize, intermediateSize)` → Float32Array
- `expertForwardBatch(hiddenIn, expertIds, expertWeights, weights, adapter, opts)` → Float32Array
- `expertReduce(hiddenOut, batchSize, topK, hiddenSize)` → Float32Array (sum top-K slots)
- `AMPLIFY_EXPERT_ID=2` marks the adapter injection expert

**Shaders copied to `shaders/`:**
- `sxme_router.hlsl`, `sxme_expert.hlsl`, `sxme_glyph_exec.hlsl`

## Notes

- sxme_glyph_exec.hlsl (INT4 glyph ISA + MoE ESC mode) has no JS equivalent — it's a pure GPU shader.
- The NumPy CPU engine (engine.py) informed the JS port: `_swiglu`, `_route_top2` match exactly.
- Adapter strength=0.10 is the default; `load_addon_shard(path, strength)` overrides it.

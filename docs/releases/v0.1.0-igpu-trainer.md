## Core innovations

### D3D11 XVM fiber model

`D3D11Engine` wraps the XVM fiber state into D3D11 UAV-backed compute buffers:

| Buffer | D3D11 type | Contents |
|--------|-----------|---------|
| `code_` | SRV (R32_UINT) | XVM bytecode (read-only) |
| `fibers_` | UAV (structured, stride=40) | XVMFiber[n] — pc/sp/phase/flags/r0-r3/entropy/pressure |
| `shared_` | UAV (R32_UINT) | Cluster shared memory |
| `stack_` | UAV | Per-fiber call stack |
| `trace_` | UAV | Execution trace ring buffer (131072 words = 512 KB) |
| `traceIndex_` | UAV | Trace write head |

Staging readback buffers for trace and traceIndex allow CPU-side trace dumping.

### SCXQ2 format parser (v1)

`src/scxq2_format.cpp` implements the wire format:
- Magic: `"SCX2"` (4 bytes)
- Header: `[version][flags][varint const_count][varint instr_count]`
- Constant pool: 9 types (string/int32/float64/false/true/null/array/object/ref)
  - All use varint length prefixes; strings are UTF-8; objects reference const pool for keys
- Instructions: `[1B packed: op=bits[5:0]][varint argc][argc×varint args][varint branch_target?]`
  - Branch target present only for ops 0x02, 0x07, 0x08
- CRC32 footer: 4 bytes little-endian over `bytes[6..instrEnd-1]`

**Ported to JS:** `src/scx/scxq2-parser.js`

### Complete D3D11 backprop pipeline

First release with a full forward + backward + optimizer cycle on GPU (cs_5_0):

**Forward:** embed → 6×[LN → QKV split → causal attn → residual → fc1 → GELU → fc2 → LN] → LN_f → LM head  
**Backward:** loss → LN_f bwd → LM head bwd → 6×[MLP bwd → attn bwd → LN bwd]  
**Optimizer:** Adam per-parameter (m_buf + v_buf + g_buf + w_buf)

Key dispatch patterns:
- `gpt2_attn_fwd.hlsl`: numthreads(1,1,1), Dispatch(seq_len,1,1) — one group per query position; local array `scores[512]` for causal mask
- `gpt2_layernorm_bwd.hlsl`: numthreads(256,1,1), Dispatch(seq_len,1,1) — two-pass reduction via `groupshared float gs_sum_dout[256]`
- `gpt2_lm_head.hlsl`: numthreads(64,1,1), Dispatch(ceil(V/64),1,1)

### DDS streaming mode

`TrainerConfig` gains optional fields:
```
shard_path    — path to .scxqdds shard file
shard_offset  — byte offset within shard
shard_size    — bytes to stream (0 = all)
use_streaming — enable DDS streaming instead of tokens.bin
```
`D3D11Engine::streamSCXQ2ToCodeBuffer()` and `streamFileSliceToCodeBuffer()` support this.

## Bug 6: Three NaN bugs (documented in docs/BUGS.md)

**Bug 6a — SRV/UAV aliasing in LN backward:** Binding `ln_out_buf_` as both SRV (t-slot, 
input x_hat) and UAV (u-slot, dhidden gradient) causes device-defined behavior. Fix: buffer copy.

**Bug 6b — SRV/UAV aliasing in GELU backward:** `ffn_pre_gelu_buf_` bound as both read source 
and gradient destination. Fix: in-place multiply workaround — accumulate into `ffn_hidden_buf_`.

**Bug 6c — LayerNorm dgamma/dbeta race:** Multiple threads writing `dgamma[i] += ...` to same 
location. Fix: strided loop `for (i = tid; i < n_embd; i += 256)` ensures each thread covers 
distinct indices, so `+=` is safe without atomics in cs_5_0.

These fixes are embedded in the shipped `gpt2_layernorm_bwd.hlsl` (two-pass reduction).

## What was NOT ported to JS

- `D3D11Engine`: native D3D11 runtime — documented in `src/d3d11/trainer-arch.js` as constants
- `GPT2Trainer`: native C++ trainer — JS equivalent is `TrainerConfig` from `src/xjsl/index.js`
- `XVMState` GPU dispatch: the GPU XVM dispatch uses `src/xvm/cpu-cluster.js` for the JS equivalent

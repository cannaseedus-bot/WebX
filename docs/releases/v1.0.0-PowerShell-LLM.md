## Key Innovations

### Semantic Reader Law Protocol
Document-as-topology — XML, TOML, CDATA, and KUHUL source are read in fixed order:

1. Preserve CDATA
2. Resolve grams
3. Enforce policy
4. Activate folds
5. Route geodesics
6. Hydrate micronauts
7. Tensors

This enforces that micronaut binding occurs before tensor dispatch, preventing authority inversion.

### tiny.x System
- Native XVM runtime written in C++
- `.brain` binary format — graph neural network weight substrate built from spectrogram Delaunay triangulation
- Hilbert-ordered node layout for cache locality
- K_experts = max(4, floor(node_count / 500)) — self-scaling expert count

### .brain Binary Format (brain1/compiler.py)
Seven binary files per brain artifact:

| File | Type | Description |
|------|------|-------------|
| tensor_header.bin | 5×uint32 LE | node_count, edge_count, csr_flat_len, K_experts, feat_dim |
| nodes_time.bin | float32[N] | temporal feature per node |
| nodes_freq.bin | float32[N] | frequency feature per node |
| nodes_energy.bin | float32[N] | energy feature per node |
| edges.bin | uint32[E×2] | src/dst pairs |
| csr_index.bin | uint32[N+1] | CSR row pointers |
| csr_neigh.bin | uint32[CSR_len] | CSR column indices |
| experts.bin | uint32[N] | KMeans cluster IDs (training) |
| routing.bin | uint32[N] | KMeans cluster IDs (inference) |

Node features are derived from ridge-detected spectrogram points ordered by Hilbert curve.

### SMCA — Structural Manifold Cluster Architecture
6-layer authority gradient — authority flows **downward only**:

```
MATRIX → CM-1 → SCXQ7 → SCXQ2 → SCO-1 → IDB
```

Rules:
- No layer may escalate authority upward
- No layer may skip a layer  
- No collapse without law

5 cluster roles: Compute, Proof, Settlement, Ledger, Relay  
6 collapse geometry classes: FOLD, CHAIN, MESH, STAR, TREE, RING

IDB is append-only, hash-addressed, causally ordered with no rollback.

### KXC IR Format
Kernel descriptor JSON with SMCA compliance metadata:
- `stackId`: "asx-xcfe-stack/v1"
- `stackCid`: FNV1A-64 over sorted artifact list
- Capability flags: needsDecompress, needsSoftmax, needsMatMul, kvInt4, isMoEKernel
- Per-entry: lawful, registryMatched, requires[], forbids[]

Artifact extensions: `.cpp`, `.hlsl`, `.wgsl`, `.cpu.cpp`, `.smca.json`

### CM-1 Dictionary
Control byte protocol for MATRIX↔SCXQ7 authority gating:

| Symbol | Byte | Meaning |
|--------|------|---------|
| SOH | 1 | Start of Header — open authority context |
| STX | 2 | Start of Text — begin payload |
| ETX | 3 | End of Text — end payload |
| EOT | 4 | End of Transmission — close authority context |
| SO | 14 | Shift Out — activate alternate control plane |
| SI | 15 | Shift In — return to default control plane |
| RS | 30 | Record Separator — boundary between kernel records |

CM-1 lane type in XJSL: `@lane:DICT`

### Shard v1.0 Binary Mapping Format
zstd+base64 encoded shards — `pack_xshard.py` for construction.

### TGUIX
PowerShell Terminal GUI for micronaut hive interaction — ISE-based, not ported to JS (PowerShell-only).

### build_tiny_x.py
Training + GGUF export pipeline for tiny.x models. Not ported (Python build tool).

## Notes

- The `.brain` format uses spectrogram-derived graph topology — the JS port exposes read/write primitives only; the Delaunay triangulation + Hilbert ordering is a Python build-time step
- SMCA is an architectural axiom set, not runnable code — the JS module exposes it as frozen constants for validation and documentation purposes
- KXC IR validator emits warnings (not errors) for MatMul kernels above the SCXQ2 layer, matching the Python reference behavior

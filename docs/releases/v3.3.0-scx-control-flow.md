## Core Innovations

### SCX Control Delta Format

Every control operation emits a delta with stable JSON + SHA256 integrity:

```js
createDelta({
  surface: 'agent',          // file|tool|command|agent|micronaut|skill|action|program|function
  target: 'agent.workflow.compound',
  operation: 'dispatch',     // invoke | load | validate | dispatch
  inputs: { workflow: 'compound.md' },
  effects: ['execute', 'memory'],  // must be in SURFACE_EFFECTS[surface]
  result: { micronauts: ['codex-agent'] },
})
```

Returns: `{ record_type, delta_id, parent_delta_id, surface, target, fold_id, authority, base_hash, operation, inputs, effects, result, replay_hash }`

`delta_id` = SHA256(stable_json_without_id). `replay_hash` = SHA256(stable_json_with_id). Enables deterministic replay and deduplication.

### Surface Authority Table

| Surface | Fold | Allowed Effects |
|---------|------|-----------------|
| file | FILE_EXEC_FOLD | read, write, execute, replay |
| tool | TOOL_EXEC_FOLD | read, write, execute, network, replay |
| command | COMMAND_EXEC_FOLD | read, write, execute, spawn, replay |
| agent | AGENT_EXEC_FOLD | read, write, execute, memory, replay |
| micronaut | MICRONAUT_EXEC_FOLD | read, write, execute, network, memory, replay |
| skill | SKILL_EXEC_FOLD | read, write, execute, memory, replay |
| action | ACTION_EXEC_FOLD | read, write, execute, replay |
| program | PROGRAM_EXEC_FOLD | read, write, execute, spawn, gpu, model, replay |
| function | FUNCTION_EXEC_FOLD | read, write, execute, gpu, model, memory, replay |

### LMStudio SDK (@lmstudio/lms-isomorphic v1.5.0)

CJS (`dist/index.cjs`) + ESM (`dist/index.mjs`) + TypeScript types. Binding factories:
- `createToolBinding({name, description, parameters, handler, source})`
- `createAgentBinding({name, model, tools, extensions, prompt, policy})`
- `createSkillBinding({name, description, command, args, verbs, tools})`
- `createCommandBinding({name, executable, args, cwd, env, verbs})`
- `createChatActBinding({model, chat, tools, structured, options})`
- `createVerbBinding({verb, surface, operation, effects, args})`
- `createExtensionBinding({name, root, manifest, kind, tools, capabilities})`

Surface-by-kind mapping: agent/chat → agent, model → micronaut, plugin/extension → program, etc.

### Compound Loop (34 workflows)

```
/explore → /specs → /plan → /work → /review → /compound → /housekeeping → repeat
```

Key workflows: explore, specs, plan, work, review, compound, housekeeping, triage, resolve_todo, changelog, heal-skill, create-agent-skill, adr, cycle (cycle detection), compound_health.

Each workflow: YAML frontmatter description + Markdown body with auto-trigger phrases, step-by-step procedure, examples.

### Agent Workflow Control

`getWorkflow(name)` → `{basename, content, mapping, delta}`. `routeWorkflow(name)` emits a micronaut dispatch delta with the workflow's mapped micronaut IDs. `workflow_mappings.json` maps workflow basenames to micronaut arrays.

## Notes

- The release uses Node.js `crypto.createHash('sha256')` for delta_id/replay_hash; the JS port uses djb2 (sync, browser-safe). For production, callers should wrap `createDelta` with a post-process step using `SubtleCrypto.digest('SHA-256', ...)`.
- `assertAllowedPath` (path traversal guard) is Node-only and not ported — browser environments don't need filesystem path authority.
- `getDependentReleaseRoots` / `getPromotionReleaseRoots` from sco-cache-index.json are runtime-only (not ported).
- LMStudio plugin root must live inside the release root — enforced in Node by `assertInsideRelease`; in the browser port this constraint is architectural (no fs access).

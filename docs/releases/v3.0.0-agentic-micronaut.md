### 3-Pass HLSL Shader Pipeline (cs_5_0 + WaveOps)

```
Pass 1: fused.hlsl      — QKV attention + force + MoE routing → emits expert_id
Pass 2: orchestrate.hlsl — compact entity worklists per expert_id
Pass 3: experts.hlsl     — dispatch one specialized kernel per ExpertId 0-7
```

cs_5_1 variants provided for Intel HD 4600 compatibility.

### 8 Expert Specializations

| ID | Name | Domain | Lane | Purpose |
|----|------|--------|------|---------|
| 0 | geometry | COGNITION | FIELD | Fold adjacency, shard geometry, DDS page layout |
| 1 | temporal | SESSION | TRACE | Session sequence, replay ordering, user-adaptive time |
| 2 | amplify | ADAPTER | DELTA | Agent/tool/command deltas without base mutation |
| 3 | compress | MEMORY | CACHE | SCXQ2 lane packing, KV quantization, DDS compaction |
| 4 | focus | GPU | FIELD | Hot GPU-resident attention folds |
| 5 | integrate | GPU | LANE | Embedding + attention + FFN + cache in one inference step |
| 6 | pattern | COGNITION | DICT | Raw-text → trained Micronaut knowledge route |
| 7 | novelty | COGNITION | EDGE | Route novelty → replay or adapter training |

### Adaptation Shader Pipeline (Reward→Mutation→Evolution)

```
Stage 1 reward.hlsl    — compute bounded reward [0,1] for adapter candidates
Stage 2 mutation.hlsl  — emit bounded adapter deltas (adapter_folds_only)
Stage 3 evolution.hlsl — select + recombine adapter candidates
```

**Invariant:** base model folds are FROZEN; only MICRONAUT_ADAPTER_FOLD is mutable.

### Coding DAG

10 nodes (acyclic), 2 terminals:

```
intent → context → plan → generate → validate → patch → test → reward → mutate → evolve → done
                                          ↓                                              ↓
                                        blocked                                       blocked
```

Each node: expert, folds, outputs. Entry: `intent`, Terminals: `done`, `blocked`.

### SCXQ2 ISA v1.2

9 opcode categories (40 opcodes), 11 domains, 11 lanes, 16 flags.

Instruction words:
- base_64: `[opcode:u8][domain:u8][fold_id:u16][offset:u32]` = 8 bytes
- extended_128: base + `[length:u16][stride:u16][lane_id:u16][flags:u16]` = 16 bytes

Principles: deterministic_execution, lawful_barriers, memory_ownership, replay_verification, entropy_bounding.

### Fold Barrier Laws

1. `session_cannot_mutate_until_replay_finalized`
2. `training_cannot_update_until_attention_verified`
3. `kv_cache_persists_with_hash`

### JSONL Coding Record Types (v3.0.0)

8 types: `coding_role_turn`, `coding_response`, `coding_dag_event`, `coding_validation`, `coding_reward`, `coding_mutation`, `coding_evolution`, `coding_replay`.

Record ID pattern: `cod_[A-Za-z0-9_-]+`. Mandatory fields: type, version, record_id, session_id, turn_id, timestamp, role, dag_node.

## Notes

- The folds/micronaut_adapters/micronaut_adapter.safetensors is excluded from the JS port (binary, ~MB range).
- 48 DDS shards (DDS_SHARD_000…047) are referenced by the geometry + compress experts but not copied (BC7, 64-tile, 4-mip — native GPU streaming only).
- Mutation policy invariant: `RECOGNITION_RULE` constant enforces that ExpertId 0-7 must each bind a fold before dispatch.
